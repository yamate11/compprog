E. Binary Strings are Simple?
time limit per test
3 seconds
memory limit per test
512 megabytes
input
standard input
output
standard output

This is an interactive problem.

You are given an unknown binary string ss∗∗ of length nn. The function f(l,r)f(l,r) is defined as the number of distinct elements in the array aa formed by the following operation:

  • Let SS be the substring slsl+1…srslsl+1…sr. Let mm be the length of the substring.
  • We start with an empty array aa.
  • For the ii-th (0≤i<m)(0≤i<m) left cyclic rotation†† of SS, let's denote the number of inversions‡‡ as xixi.
  • For every ii such that 0≤i<m0≤i<m, append xi mod mxi mod m to the array aa, where u mod vu mod v denotes the remainder of dividing uu by vv.

To determine ss, you can ask some questions. In each question, you can choose 22 integers ll and rr (1≤l≤r≤n1≤l≤r≤n) and get the value of f(l,r)f(l,r). Asking each query incurs a cost of nr−l+1nr−l+1. Note that the cost does not necessarily have to be an integer.

You have to determine the hidden binary string while keeping the total cost of queries atmost max(30,3⋅n)max(30,3⋅n). You are allowed to make at most 22 guesses.

∗∗A binary string only contains characters 00 and 11.

††Let there be a binary string s=s1s2⋯sns=s1s2⋯sn. The kk-th left cyclic rotation of ss is defined as tk=sk+1sk+2⋯sns1s2⋯sktk=sk+1sk+2⋯sns1s2⋯sk.

‡‡Let there be a string s=s1s2⋯sns=s1s2⋯sn. The number of inversions of ss is defined as the number of pairs of indices i,j(1≤i<j≤n)i,j(1≤i<j≤n), such that si>sjsi>sj.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1001≤t≤100). The description of the test cases follows.

The first line of each test case contains a single integer nn (2≤n≤30002≤n≤3000) — the length of the unknown string ss.

It is guaranteed that the sum of nn over all test cases does not exceed 30003000.

Interaction

To ask a question, output a line in the following format (without quotes)

  • "? l r? l r" (1≤l≤r≤n1≤l≤r≤n)
The jury will return an integer f(l,r)f(l,r).

For guessing a string ss (length of ss must be nn), output a single line in the following format (without quotes)

  • "! s! s"
The jury will return 1 if the guess is correct or 0 if the guess is incorrect.

Guessing a string does not count as a query.

After that, if your string is correct, proceed to the next test case or terminate the program if it was the last test case.

The interactor is not adaptive, meaning that the answer is known before the participant asks the queries and doesn't depend on the queries asked by the participant.

After printing each query do not forget to output the end of line and flush∗∗ the output. Otherwise, you will get Idleness limit exceeded verdict.

If, at any interaction step, you read −1−1 instead of valid data, your solution must exit immediately. This means that your solution will receive Wrong answer because of an invalid query or any other mistake. Failing to exit can result in an arbitrary verdict because your solution will continue to read from a closed stream.

Hacks

To make a hack, use the following format.

The first line should contain a single integer tt (1≤t≤1001≤t≤100) — the number of test cases.

The first line of each test case should contain nn (2≤n≤30002≤n≤3000), the length of ss.

The second line of each test case should contain ss, a binary string of length nn.

The sum of nn over all test cases should not exceed 30003000.

∗∗To flush, use:

  • fflush(stdout) or cout.flush() in C++;
  • sys.stdout.flush() in Python;
  • see the documentation for other languages.

In the first test case, the hidden string is s=s= 00000. Here, since 0 cannot create any inversion, the array aa will always be [0][0] for all substrings. Hence f(l,r)f(l,r) returns 11 for all queries.

Here we made 33 queries.

The first query is f(1,5)f(1,5) which returns 1 — the cost of this query is 55−1+1=155−1+1=1.

The second query is f(2,5)f(2,5), which returns 1 — the cost of this query is 55−2+1=5455−2+1=54.

The third query is f(3,5)f(3,5), which returns 1 — the cost of this query is 55−3+1=5355−3+1=53.

In the second test case, the hidden string is s=s= 001.

We make the query f(1,3)f(1,3), which returns 3. The cost of this query is 33−1+1=133−1+1=1.

The total cost of the query is 11 which is less than max(30,3⋅n)=30max(30,3⋅n)=30.

We then guess 100, for which the interactor returns 0, indicating it is incorrect.
