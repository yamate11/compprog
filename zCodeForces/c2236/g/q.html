G. Criterion in Burlandia
time limit per test
3 seconds
memory limit per test
512 megabytes
input
standard input
output
standard output

Regions in Burlandia form a graph with nn vertices and n−1n−1 edges, and there is exactly one path between any two vertices. Formally, the regions form a tree.

Each region has a friendliness value aiai.

There are qq queries. In each query, a pair of friends located in different regions is given. They want to know how many subsegments of the path between these regions are hospitable.

It is known that in Burlandia there are two criteria for evaluating relationships — XOR and sum. A subsegment of the path, containing some vertices lying on the path from region xx to region yy, is called hospitable if it is non-empty and the sum of friendliness values on this subsegment does not exceed their XOR.

More formally, for each query you are given two vertices xx and yy (x≠y)(x≠y). Consider the shortest path from vertex xx to vertex yy in the tree. Let the vertices v1,v2,…,vkv1,v2,…,vk form this path, where v1=xv1=x, vk=yvk=y. You need to find the number of subsegments of this path for which the following condition holds:

avl⊕avl+1⊕…⊕avr≥(avl+avl+1+…+avr),avl⊕avl+1⊕…⊕avr≥(avl+avl+1+…+avr),
where 1≤l≤r≤k1≤l≤r≤k — the boundaries of the subsegment of vertices on the path from xx to yy.

Input

Each test consists of multiple test cases. The first line contains a single integer tt (1≤t≤1041≤t≤104) — the number of test cases. Then the descriptions of the test cases follow.

The first line of each test case contains integers nn (2≤n≤1052≤n≤105) — the number of vertices in the tree, and qq (1≤q≤1051≤q≤105) — the number of queries.

The second line contains an array of nn non-negative integers — the friendliness values of the regions (0≤ai<2200≤ai<220).

The next n−1n−1 lines describe the edges of the tree: each line contains integers u,vu,v (1≤u,v≤n1≤u,v≤n) — an edge.

Then qq lines follow describing the queries. Each query is given by integers x,yx,y (1≤x,y≤n1≤x,y≤n, x≠yx≠y) — the vertices that define the path for which you need to count the number of hospitable subsegments.

It is guaranteed that the sum of nn and the sum of qq over all test cases do not exceed 105105, and that the edges indeed form a tree.

Output

For each query output an answer on a separate line.

Note

For clarity, consider the third query from the third sample.

The path from vertex 22 to vertex 33 consists of vertices 2,4,32,4,3.

The subsegment [2;3][2;3] does not satisfy the condition, since the XOR on it is a4⊕a3=4⊕4=0a4⊕a3=4⊕4=0, while the sum is a4+a3=4+4=8a4+a3=4+4=8.

The subsegment [1;3][1;3] does not satisfy the condition, since the XOR on it is a2⊕a4⊕a3=2⊕4⊕4=2a2⊕a4⊕a3=2⊕4⊕4=2, while the sum is a2+a4+a3=2+4+4=10a2+a4+a3=2+4+4=10.

It can be shown that all other 4 subsegments satisfy the condition.

For each query, output the answer in a separate line.
